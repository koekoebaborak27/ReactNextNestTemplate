# 13 手動ブラウザ確認手順

## 概要
Claude Codeによる自動確認（11_operation_check.md）とは別に、
開発者自身がブラウザで目視確認するための手順書です。

---

## 事前準備

### 1. 起動確認
以下の2つのターミナルを開いて、それぞれ起動してください。

**ターミナル1（バックエンド）**
```bash
cd backend
npm run start:dev
```
以下のメッセージが表示されれば起動成功です。
Application is running on: http://localhost:3001

**ターミナル2（フロントエンド）**
```bash
cd frontend
npm run dev
```
以下のメッセージが表示されれば起動成功です。
Local: http://localhost:3000

---

## 確認手順

### ✅ 確認1: ログイン画面
1. ブラウザで http://localhost:3000 を開く
2. 以下を目視確認する
   - 「地図販売システム」というタイトルが表示されている
   - メールアドレス入力欄がある
   - パスワード入力欄がある
   - ログインボタンがある

---

### ✅ 確認2: ログイン失敗時のエラー表示
1. メールアドレスに「 wrong@example.com 」と入力する
2. パスワードに「 wrongpassword 」と入力する
3. ログインボタンをクリックする
4. 以下を目視確認する
   - 「メールアドレスまたはパスワードが違います」というエラーメッセージが表示される
   - 画面遷移しない

---

### ✅ 確認3: ログイン成功・会社一覧画面への遷移
1. メールアドレスに「 admin@example.com 」と入力する
2. パスワードに「 password123 」と入力する
3. ログインボタンをクリックする
4. 以下を目視確認する
   - 契約会社一覧画面（/companies）に遷移する
   - 「株式会社地図太郎」「有限会社マップ商事」の2社が表示されている

---

### ✅ 確認4: 会社名検索
1. 検索欄に「　地図　」と入力してボタンをクリックする
2. 以下を目視確認する
   - 「株式会社地図太郎」のみ表示される
3. 検索欄を空にして再検索する
4. 以下を目視確認する
   - 2社とも表示される

---

### ✅ 確認5: 会社詳細画面
1. 「株式会社地図太郎」をクリックする
2. 以下を目視確認する
   - 会社情報（会社名・担当者名・住所）が表示されている
   - 契約一覧に「道路地図」「観光地図」の2件が表示されている
   - 「請求書を発行する」ボタンがある

---

### ✅ 確認6: 未認証時のリダイレクト
1. ブラウザのlocalStorageからaccessTokenを削除する
   - 開発者ツール（F12）→「Application」→「Local Storage」→「http://localhost:3000」→「accessToken」を右クリックして削除
2. ブラウザで http://localhost:3000/companies を直接開く
3. 以下を目視確認する
   - ログイン画面（/）にリダイレクトされる

---

### ✅ 確認7: 請求書発行
1. 再度ログインして「株式会社地図太郎」の詳細画面を開く
2. 「請求書を発行する」ボタンをクリックする
3. 以下を目視確認する
   - 請求書発行画面（/companies/1/invoices/new）に遷移する
   - 契約のプルダウンに「道路地図」「観光地図」が表示されている
4. 「道路地図」を選択し、単価に「1000」と入力する
5. 以下を目視確認する
   - 印税額プレビューに「¥50,000」が表示される
   - （計算式: 数量1000 × 単価1000円 × 印税率0.05 = 50,000円）
6. 「請求書を発行する」ボタンをクリックする
7. 以下を目視確認する
   - 会社詳細画面に戻る
   - 請求書一覧に発行した請求書が「承認待ち」で追加されている
   - 金額が「¥50,000」になっている

## トラブルシューティング

### バックエンドに接続できない場合
- backendが起動しているか確認する（ターミナル1）
- http://localhost:3001 にブラウザで直接アクセスして応答があるか確認する

### ログインできない場合
- シードデータが投入済みか確認する（10_seed_data.md参照）
- 投入されていない場合はbackendディレクトリで `npx prisma db seed` を実行する

### 画面が真っ白になる場合
- ブラウザの開発者ツール（F12）のConsoleタブでエラーを確認する
- frontendが起動しているか確認する（ターミナル2）
